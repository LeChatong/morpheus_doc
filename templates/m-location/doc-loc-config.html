Ce chapitre concerne les configurations, nous allons par la suite présenter dans les moindres détails comment utiliser cette partie.

{{ 'CONFIGURATION CITéS'|upper }}

Il s’agit ici des configurations permettant la création (de Virtualiser) des cités universitaire existante.

Interface de Configuration des Cités
(1) Création d’une cité universitaire
Fenêtre d'édition d'une Cité

Une cité universitaire est un bâtiment abritant des locaux ou plus précisément des chambres d’étudiant parfois meublés et généralement sur la supervision d’un concierge. La création des cités universitaires est le point d’entrée de ce module. Le formulaire si dessus permet l’ajout d’une cité universitaire. Il faut spécifier le nom de la cité (libellé) et sélectionner le concierge de la cité précédemment créé.

(2) Manipulation des cités

Le groupe d’action présenté dans le (2) permet l’édition, c’est-à-dire la modification du libellé ou du concierge et la suppression.

(3) Création de concierge
Interface de Configuration des Concierges

L’onglet (3) renvoie sur la page de gestion des concierges. En cliquant sur le bouton (3.1) on obtient le formulaire de création d’un concierge si dessous.

Fenêtre d'édition d'un Concierge

Lorsque le formulaire s’affiche il suffit de remplir le nom (* devant le champ nom signifie que le nom doit être obligatoirement fourni), le prénom, la date de naissance et le téléphone du concierge du concierge et cliquer sur enregistrer pour terminer la création d’un concierge.

{{ 'CONFIGURATIONS DES MEUBLES'|upper }}

Les meubles sont des mobiliers présents dans le local lors de l’entrée de l’étudiant. Un dommage sur ces meubles lors de la sortie de l’étudiant entrainera des frais de réparations qui seront prélevés sur la caution de l’étudiant.

Interface de Configuration des meubles
(1) Création de meubles
Fenêtre d'édition d'un meuble

En cliquant sur le bouton (1) on obtient le formulaire ci-dessus qui permet d’ajouter un meuble à un local d’une cité. Il suffit de remplir le libellé du meuble, le prix en cas de dommage et le type de meuble correspondant.

(2) Création des types de meubles
Interface de Configuration des types de meubles

En cliquant sur l’onglet (2) on a accès à la gestion des types de meuble. Un type de meuble représente la nature d’un meuble (table, chaise, etc.). Le bouton (2.1) permet de lancer le formulaire si dessous qui permet la création des types de meubles.

Fenêtre d'édition d'un type de meuble

Le formulaire lancé, il suffit remplir le libellé du type de meuble et d’enregistrer.

{{ 'CONFIGURATIONS SERVICES'|upper }}

Dans le module location le service représente des éléments sur lesquels le système s’appuie pour générer les factures de paiement d’un locataire.

Interface de Configuration des Services
(1) Création de services
Fenêtre d'édition d'un service

En cliquant sur le bouton on obtient se formulaire qui permet la création d’un service. Le formulaire lancé, il suffit de remplir le libellé du service, spécifier si le service est actif (un service inactif ne peut pas être utilisé) et ensuite spécifier si le service doit être payé avant d’être utilisé puis cliquer sur enregistrer.

{{ 'CONFIGURATIONS DE TEMPLATES DE LOCAL'|upper }}

Les Templates représentent les types de locaux qu’on peut retrouver dans une cité (studios, chambre, appartement), leurs structures internes (l’ensemble des pièces), leurs contenus (meubles) et leurs prix.

Interface de Configuration des Templates
(1) Création de Template
Fenêtre d'édition d'un Template

En cliquant sur le bouton (1) (Nouveau) on obtient le formulaire si dessus qui permet l’ajout d’un Template. Le Template à sa création est vide nous verrons comment le remplir dans les paragraphes suivant.

(2) Lister me contenu d’un Template

Le bouton (2) (choisir) permet de liste le contenu d’un Template (pièces, services et prix, meubles).

Détails sur un Template
(3) Ajouter des pièces à un Template
Fenêtre d'ajout d'une pièce à un Template

En cliquant sur le bouton (3) (Nouveau) on obtient le formulaire si dessus qui permet d’ajouter une pièce à un Template en sélectionnant la pièce et en indiquant le nombre de pièce du type qu’on retrouve dans ce Template.

(4) Ajouter des services à un Template
Fenêtre d'ajout d'un service à un Template

En cliquant sur le bouton (4) (Nouveau) on obtient le formulaire si dessus qui permet d’ajouter un service à un Template en sélectionnant le service et en indiquant le montant annuelle pour le service.

NB : le champ montant ajouter au formulaire d’ajout d’un service au template est une particularité de la gestion des cités universitaires où les étudiants payent les services de façon annuelle. Cette configuration évite d’aller configurer de façon particulière les services pour chaque local d’étudiant.

(5) Ajouter des meubles à un Template
Fenêtre d'ajout d'un service à un Template

En cliquant sur le bouton (5) (Nouveau) on obtient le formulaire si dessus qui permet d’ajouter un meuble à un Template en sélectionnant le type de meuble et en indiquant le nombre de meuble du type qu’on retrouve dans ce Template.

(6) Création des pièces
Interface de Création des Pièces

En cliquant sur l’onglet (6) on obtient la page de gestion des type de pièce en cliquant sur le bouton (6.1) on lance la fenêtre de création d’un type de pièce si dessous.

Fenêtre d'Ajout d'une pièce

Il suffit de remplir le libellé du type de meuble et de cliquer sur enregistrer.

{{ 'CONFIGURATIONS LOCAUX DE CITE'|upper }}

Ce sont les locaux de cité qui font l’objet des contrats de location. C’est un local auquel a été attribué un Template (pièces, meubles, service) qui est loué à un locataire dans notre cas un étudiant.

Interface de Configuration des Locaux
(1) Création d’un local
Fenêtre d'Ajout d'un locak

Après avoir sélectionné la cité dans la quel on veut créer le local, on clique sur le bouton (1) (Nouveau) et on obtient le formulaire ci-dessus qui permet d’ajouter un local à un cité. On spécifie le nom du local, on sélectionne le statut du local (toujours disponible lors de la création), on sélectionne le Template de la chambre puis on clique sur enregistrer.

(2) Créer un statut de local

Les statuts permettent de donner l’état de disponibilité d’un local. Elle peut prendre plusieurs valeurs selon la configuration mais les deux valeurs de base son « disponible » et « occupé » en cliquant sur l’onglet (2) on obtient la page de gestion des statuts suivante.

Interface de Configuration des statuts de locak

Pour créer un statut, il faut cliquer sur le bouton (2.1) qui lance le formulaire de création d’un statut suivant :

Fenêtre d'édition d'un statut

Le formulaire lancé, il ne reste plus qu’à créer le statut et à cliquer sur enregistrer.
